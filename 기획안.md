# 플레이영 용인구갈점 — 웹 고객관리 프로그램 최종 기획안

> 작성일 2026-06-10 · 키즈카페 스탬프/쿠폰 고객관리 웹앱

---

## 1. 목표
매장 카운터에서 태블릿·휴대폰으로 접속해, 전화번호 기반으로 **스탬프 적립**과 **쿠폰 사용**을 처리하고
**고객 목록을 관리**하는 무료 웹 프로그램.

## 2. 데이터 모델

새 고객관리 데이터는 **전화번호 1개 = 고객 1명** 기준의 단일 테이블로 구성한다.

| 필드 | 타입 | 설명 |
|---|---|---|
| `phone` | 문자열(숫자만) | 전화번호. 고유값(중복 불가). 앞자리 0 보존 |
| `stamps` | 정수 | 스탬프 잔여값 (0~9, 10이 되면 쿠폰으로 전환) |
| `coupons` | 정수 | 보유중인 쿠폰개수 |
| `coupons_used` | 정수 | 사용한 쿠폰개수(누적) |
| `name` | 문자열 | 이름 (선택, 카운터 식별용) |
| `memo` | 문자열 | 메모 (선택) |

### 2.1 최초 데이터 마이그레이션 (엑셀 → DB)
원본: `20260430_플레이영 용인구갈점_고객정보.xlsx` (11,605행, '전화번호'·'스탬프 잔여값'·'쿠폰명' 사용)

1. **중복 전화번호 합산** — 같은 번호의 모든 행 스탬프를 더한다. (중복 2,022건)
2. **쿠폰 전환** — 합산 스탬프를 10으로 나눠 몫만큼 쿠폰 부여, 나머지를 스탬프 잔여값으로 둔다.
   (마이그레이션 시점에도 동일 규칙 적용)
3. **기존 쿠폰** — 엑셀 '쿠폰명'(`1시간 무료이용권`)이 채워진 고객은 보유 쿠폰 **+1**. 종류 구분 없이 하나로 통합.
4. **사용한 쿠폰개수** — 0으로 시작.

> **변환 결과(검증 완료):** 고유 고객 **9,583명**, 보유 쿠폰 고객 **270명**, 마이그레이션 후 최대 스탬프 9.

## 3. 동작 규칙

### 3.1 스탬프 적립
- 전화번호 + 적립할 스탬프 수 입력 → 해당 고객 스탬프에 더한다.
- 더한 결과가 **10 이상이면 10개마다 쿠폰 1개**로 전환하고 나머지를 잔여값으로 둔다.
  - 예: 잔여 8 + 적립 15 = 23 → **쿠폰 +2, 스탬프 잔여 3**
- 없는 전화번호면 새 고객으로 자동 생성 후 적립.

### 3.2 쿠폰 사용
- 전화번호 입력 → 현재 보유 쿠폰개수 표시.
- "사용하기" → 보유 쿠폰 **-1**, 사용한 쿠폰개수 **+1**. (보유 0이면 사용 불가)

### 3.3 고객 관리
- 고객 목록을 **전화번호로 검색**, **페이지네이션**으로 표시.
- 데이터 **직접 추가 / 수정 / 삭제** 가능.

## 4. 화면 구성 (태블릿 우선, 큰 버튼·반응형)

1. **로그인** — 비밀번호 입력 후 진입.
2. **스탬프 입력** 탭 (기본 화면)
3. **쿠폰 사용** 탭
4. **고객 목록** 탭 — 검색 / 페이지네이션 / 추가·수정·삭제

## 5. 기술 구성 (전부 무료)

| 구성 | 사용 기술 | 비고 |

## 6. 보안 — "간단한 서버 비밀번호"
- 로그인 시 입력한 비밀번호(`play1122#`)로 Supabase Auth 단일 계정에 로그인한다.
- DB는 **RLS(행 수준 보안)** 로 잠가 **로그인하지 않으면 어떤 데이터도 읽거나 쓸 수 없다.**
- 즉, URL을 알아도 비밀번호 없이는 고객 전화번호 데이터에 접근 불가.
- 프론트엔드의 Supabase anon key는 공개되어도 안전한 설계(공개용 키).
